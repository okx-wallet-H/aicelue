# 交易日记

## 2026-04-17 凌晨到上午策略复盘与改造记录

### 交易数据统计

今天凌晨先对实盘数据进行了集中复盘。当前样本共 **99 笔交易**，其中盈利 **43 笔**、亏损 **56 笔**，胜率 **43.4%**。累计总盈利约 **281U**，总亏损约 **460U**，净亏损约 **178U**。平均每笔盈利约 **6.54U**，平均每笔亏损约 **8.22U**，盈亏比只有 **0.80**。从账户路径上看，账户资金已经从 **1500U** 回撤到约 **1332U**，说明当前问题并不在于完全没有盈利能力，而在于盈利释放不足、亏损控制失衡，最终被几次大额回撤和频繁鞭锯吞噬。

### 核心问题判断

本轮分析确认，当前策略的两个最核心问题分别是 **大额亏损** 与 **震荡市鞭锯**。前者导致少数单笔损失快速吃掉多笔小盈利，后者则使高频追趋势策略在 SOL 上不断来回止损。与此同时，策略原先还存在若干执行层缺陷，包括保证金利用率偏低、信号通过后二次拦截、止盈止损表达方式不统一，以及仓位与资金分配不够贴近比赛冲刺目标。换言之，问题并不是单一指标失效，而是“信号生成、资金利用、执行落地、盘中风控”四个环节同时存在磨损。

| 维度 | 发现的问题 | 直接影响 |
|---|---|---|
| 盈亏结构 | 大额亏损过多，平均亏损高于平均盈利 | 少数错误单即可吞掉连续盈利 |
| 市场适配 | 震荡市仍高频追趋势，尤其 SOL 容易鞭锯 | 胜率下降，连续止损增多 |
| 执行逻辑 | 信号通过后仍被保守分支二次拦截 | 错失高质量进场机会 |
| 资金利用 | 保证金、下单量、资金分配逻辑偏保守 | 比赛阶段资金效率不足 |

### 今天凌晨上线的高频比赛模式

凌晨首先完成了高频比赛模式的第一轮提速改造。核心思路是面向比赛冲刺目标，适度降低入场阻力、提高资金效率，并允许策略在确认方向后更快完成开仓、同向加仓或反手切换。这里的重点不是无条件增加交易次数，而是在高频轮询框架内减少不必要的保守层，让有效信号可以真正落地。

本轮改动中，高频模式的主要特征包括：降低信号门槛、提高常规杠杆档位、允许同向加仓和反向先平后开，并强化 LLM 对短线节奏的直接影响。此前系统更像“先产生机会、再自己否决机会”，而高频模式的目标是把明显成立的交易信号快速转化为执行动作，避免因层层拦截而失去比赛期最重要的时效性。

### 资金利用率优化

随后对资金利用率进行了系统修正。第一项是把最小可执行保证金放宽到 **100U** 级别，以减少因单标的资金门槛过高导致的机会浪费。第二项是把下单数量改为按 **USDT 名义价值** 反算合约张数，而不再依赖容易偏差的简单估算。第三项是在执行前增加了 OKX 的可下单量校验，形成“本地估算 + 交易所上限回查”的双保险机制，尽量避免 `51008` 类保证金不足或张数超限错误。

这一轮修改的直接目的，是让系统在资金偏紧、比赛又强调资金效率时，依然能尽可能把有效仓位送达交易所，而不是频繁卡死在参数边界上。对于短周期策略而言，能否稳定成交，本身就是收益曲线的一部分。

### 二次拦截修复

凌晨阶段还修复了策略引擎中的“二次拦截”问题。此前已经出现多次这样的情况：大模型已经给出明确方向，量化信号分也通过了门槛，但在更后面的保护分支里又被重新否决，结果就是“明明该买却没买，明明该卖却没卖”。

本次修复的原则非常明确：**在基础门槛已经通过、且不存在硬风控阻断时，大模型明确支持的方向应该真正执行**。这次改动等于把原来“过度保守的重复审核”从链路里拆掉，让上层决策与底层执行的关系更加一致。

### 止盈止损调整

为了统一策略语言并提升盘中管理的灵活性，今天又把止盈止损的表达改成了 **百分比制**。这样一来，止盈止损不再依赖固定价差或离散数值，而是可以随价格水平与标的波动自动保持相对一致的风险刻度。

在此基础上，又引入了 **阶梯式移动止盈**。新逻辑会记录持仓的峰值浮盈，并按浮盈区间逐级抬高锁利线。也就是说，持仓一旦跑出利润，系统不会再只是被动等待固定止盈或止损，而是会随着盈利扩大逐步锁住更多已得利润，避免原本已经拿到的浮盈在震荡回撤中大幅回吐。

| 浮盈阶段 | 锁利目标 | 动作含义 |
|---|---:|---|
| 2% | 锁定 1% | 先保住最基本盈利 |
| 3% | 锁定 2% | 防止小波动回吐全部利润 |
| 5% | 锁定 3% | 趋势确认后加快利润保护 |
| 8% | 锁定 5% | 中段利润开始实质落袋 |
| 10% | 锁定 7% | 冲刺利润优先保住大头 |

### LLM 仓位决策上线

在仓位层面，今天也完成了 **LLM 仓位决策** 的接入。新的逻辑允许大模型在 `trade_advice` 中直接返回 `position_pct`，系统会优先采纳这个仓位建议，并限制在合理区间内执行；如果模型没有给出仓位，则再回退到原有 Kelly 计算。这样做的意义在于，仓位不再只由历史统计参数驱动，而是可以结合当下市场结构、方向清晰度和短线赔率，由模型直接参与实时调节。

同时，开仓保证金也改成按 **固定本金 1500U × position_ratio** 计算，并设置最低保证金门槛，以保证仓位信号能稳定落到真实下单数量，而不是被账户权益的短时变化过度扰动。

### 风控放宽与亏损上限调整

针对之前“过早熔断、被动错失后续修复机会”的问题，今天还对总风控做了放宽。日内熔断阈值已经放宽到 **15%**，单笔亏损预算也调整为按本金 **3%** 计算，以便在比赛环境下兼顾生存与进攻。这里的逻辑不是放弃风控，而是承认比赛冲刺期需要更高的风险承载能力，同时通过更快的止损与更严格的盘中过滤，避免风险完全依赖静态熔断来处理。

### 智能过滤优化

上午阶段，针对 SOL 在震荡市的反复止损问题，重点加入了更强的市场过滤。当前系统已引入 **趋势 / 震荡识别**：当 ADX 明确偏高时，才允许按趋势逻辑跟随交易；当 ADX 偏低时，则不再追涨杀跌，而是仅在价格靠近布林带上下轨时考虑均值回归，否则直接观望。

与此同时，又增加了 **RSI 过滤**。在多头方向，RSI 过热时不再继续追多，要等待回调后再进场；在空头方向，RSI 过冷时不再继续追空，要等待反弹后再重新评估。若已有仓位且 RSI 已进入明显过热区，则会优先触发减仓保护，避免利润在高位钝化后被快速回吐。

信号门槛方面，也已经从过于激进的低门槛回调到更稳妥的 **0.28** 中心值，以减少“每轮都想出手”的冲动倾向，把交易频率留给更有把握的单子。

### 今日改动总览

今天凌晨到上午，已经连续完成以下几类关键更新，整体方向可以概括为：**减少无效出手、提高资金效率、让强信号真正执行、把利润保护做得更主动**。

| 模块 | 今天完成的改动 |
|---|---|
| 高频模式 | 降低门槛、提高杠杆、允许加仓与反手 |
| 执行层 | 最低保证金优化、按 USDT 反算张数、OKX 双保险 |
| 决策链路 | 修复二次拦截，大模型说买就买 |
| 止盈止损 | 切换为百分比制，加入阶梯移动止盈 |
| 智能过滤 | 趋势/震荡识别、RSI 过滤、信号门槛回调至 0.28 |
| 风控 | 熔断放宽到 15%，单笔亏损改为本金 3% |
| 仓位管理 | LLM 仓位决策上线 |

### 阶段结论

截至今天上午，策略的修正方向已经非常明确：后续不再追求“所有行情都参与”，而是优先解决 **大亏损、鞭锯、执行打折、资金利用不足** 四个问题。若后续数据能验证震荡过滤与利润保护确实生效，则下一阶段重点应放在继续压缩 SOL 的无效交易、把资金更多集中到最强信号上，并让策略在比赛后段具备更强的净值弹性。

当前结论是，系统并非没有赚钱能力，而是此前把太多收益让渡给了错误的交易环境和不够果断的执行链路。今天这一轮修改，目标正是把这两部分损耗尽快补回来。
